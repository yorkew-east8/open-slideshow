# 用例（Use Cases）

> 用例编号 UC-xx，前置/主流程/后置条件。覆盖功能要求 1–4。

## UC-1 首次启动进入首页

- **前置**：容器启动，浏览器访问 `http://localhost:8080`。
- **主流程**：
  1. 首页加载，前端读 `localStorage` 历史。
  2. 首次为空 → 左侧历史列表显示空状态提示（“还没有播放过的幻灯片”）。
  3. 右侧风格下拉默认选中 `DEFAULT_THEME`（官方默认）。
  4. 底部“开始播放”按钮可见但首次点击需先选定文件（见 UC-3 的禁用态）。
- **后置**：用户停留在首页，准备选片。

## UC-2 打开新文件并校验（需求 1.2）

- **前置**：用户在首页。
- **主流程（合法文件）**：
  1. 点“打开新文件” → 触发 `<input type=file accept=".md">`。
  2. 选择本地 `.md` → 前端 `validate.ts` 校验：
     - 合法条件：含至少一个 `---`（frontmatter 闭合后的分页符），或 frontmatter 含 `theme`/`layout`。
  3. 校验通过 → 上传 `POST /api/files` → server 写入 `user-slides/<原名>`。
  4. 加入历史（`openedAt=now`），列表置顶。
- **主流程（非法文件）**：
  1. 同上选择。
  2. 校验失败 → 弹出原因（例如“未检测到 `---` 分页符，可能不是幻灯片”）。
  3. 提供「强制打开」按钮 → 用户确认 → 走合法流程继续。
- **后置**：该文件出现在历史列表，可被激活播放。

## UC-3 从历史列表选片并开始播放（需求 1.1、1.3、3）

- **前置**：历史列表至少 1 项（或刚 UC-2 打开过）。
- **主流程**：
  1. 用户在左侧历史列表点某文件 → 高亮选中。
  2. 右侧选风格（可选，默认官方）。
  3. 底部“开始播放”按钮变为可用 → 点击。
  4. 前端 `POST /api/activate { filename, theme }` → server 把目标写入 `active.md`（注入 frontmatter theme）。
  5. 前端 `window.open('http://localhost:3030', '_blank')` 新标签播放。
  6. 该文件 `openedAt=now`、`timesOpened++`，历史置顶。
- **后置**：Slidev 渲染选中文件；历史已更新。

## UC-4 播放结束返回首页（需求 4，调整为新标签方案）

- **前置**：在 `:3030` 播放中（新标签页打开）。
- **主流程**：
  1. 演讲结束，关闭播放标签页。
  2. 首页标签页（`:8080`）仍在，自动回到首页状态。
- **后置**：回到首页。玩家不注入任何自定义按钮，保持官方纯净。
- **设计决策**：取消”返回首页”按钮 —— 玩家以官方代码渲染，避免自定义全局层带来的升级耦合。

## UC-5 切换风格（需求 2）

- **前置**：首页。
- **主流程**：
  1. 右侧风格下拉列出已安装主题（`GET /api/themes`）。
  2. 选择 → 仅记录到前端状态（不立即改文件）。
  3. 在 UC-3 激活时一并注入 `active.md` 的 frontmatter。
- **后置**：播放使用所选主题。

## UC-6 历史列表排序与去重（需求 1.1）

- **规则**：按 `openedAt` 倒序；同文件名去重，更新时间并累加 `timesOpened`。
- **上限**：`HISTORY_LIMIT`（默认 20），超出按最旧剔除。

## 约束与边界

- 浏览器无法读任意本地路径 → 文件必须先上传落地到 `user-slides/`（见 architecture.md §2）。
- 非法文件强制打开：仍写入并播放，前端仅给出风险提示，不阻断。
